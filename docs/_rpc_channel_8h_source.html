82// Abstract interface for an RPC channel. An RpcChannel represents a
83// communication line to a Service which can be used to call that Service's
84// methods. The Service may be running on another machine. Normally, you
85// should not call an RpcChannel directly, but instead construct a stub Service
86// wrapping it. Example:
87// FIXME: update here
88// RpcChannel* channel = new MyRpcChannel("remotehost.example.com:1234");
89// MyService* service = new MyService::Stub(channel);
90// service->MyMethod(request, &response, callback);

111 // procedure looks the same as Service::CallMethod(), but the requirements
112 // are less strict in one important way: the request and response objects
113 // need not be of any specific class as long as their descriptors are
114 // method->input_type() and method->output_type().
